5月29日考试总结

A

题目传送门

思路

模版题，直接套模版就可以了，就是把加法改成了异或。

时间复杂度：

  • 树链剖分 $\mathcal O (n)$

  • 线段树 $\mathcal O (n \log n)$

  • 总时间复杂度 $\mathcal O (n \log n)$

代码

B

题目传送门

B 题在考试里尝试了一种新的染色做法，考场里没写出来。这种做法虽然可以做，但是做法有许多很累赘的地方，所以这种做法不太好。

思路

思路就是我们树剖完后，用线段树记录在这段区间内最深的那个点就行。

每次跟新的时候，就直接单点更新就行。

注意初始值要赋值成 $-1$。

时间复杂度：

  • 树链剖分 $\mathcal O (n)$

  • 线段树 $\mathcal O (n \log n)$

  • 总体时间复杂度 $\mathcal O (n \log n)$

代码

C

题目传送门

这道题丢分好可惜。

思路

我们发现，如果我们的替换道路必须要在一个环上，不然的话，条道路就是一个废物，没有任何的用处。

所以，这条边能产生贡献的就是他们的路径就是他们的一个环的路径

借用下洛谷的图片：）

我们发现我们要跟新的就是这章图的蓝边，而这个蓝边就是 12 ~ 5 的树上路径

于是我们就边权传点权，把这个预备边的值都赋值到边上，然后求解最小值。

然后在求解答案的时候，我们去这两条边 $dep$ 大的那一个点，将他作为答案。在 query 函数的内部去最小值就是这条边废掉时的答案。

为什么要取 $dep$ 大的点呢？其实这和你写的边权转点权的写法有关系，我的写法是将其向上转。

但这还有一个问题，就是如何保证覆盖的是最小的呢，这个简单，我们按照预备边的权值排个序就行。

考试的时候，我写线段树时，我下传懒标记的时候是给下一层下传的，然后就出现了问题。

时间复杂度：

  • 树链剖分 $\mathcal O (n)$

  • 排序 $\mathcal O (n \log n)$

  • 线段树 $\mathcal O (n \log n)$

  • 总体时间复杂度 $\mathcal O (n \log n)$，常数稍大
