A

题目大意

给定 $n$ 个朋友，每人有消费金额 $x_i$ 和预算金额 $y_i$。要求将这些朋友分成若干组（每组至少两人，且每人只能属于一组），使得每组满足 $\sum x_i \leq \sum y_i$。求最多能分成多少组。

（注：目标是最大化组数，而非最大化参与人数）

思路

我们直接贪心，我们对 $y_i - x_i$ 从小到大排序，然后我们根据贪心策略，我们要拿 $y_i - x_i$ 最大的那个和 $y_i - x_i$ 最小的那个人匹配，这样才能发挥出这个大块头的最大的价值。

B

题目大意

给定一棵 $n$ 个节点的树（$1$ 号节点为根），每个节点初始有 $1$ 个苹果。每秒所有苹果（除根节点外）向父节点移动 $1$ 步，到达根节点的苹果被立即收集。若同一节点在同一时刻有多个苹果，则它们会成对湮灭（奇数剩 $1$，偶数剩 $0$）。求最终能收集到多少苹果。

（注：问题转化为统计每个节点的深度，并计算所有深度的奇偶贡献）

思路

傻瓜题目，就是题目有点难看懂，如果看懂了直接模拟即可。

C

题目大意

给定一个长度为 $n$ 的正整数序列 $a$，选择两个不重叠的连续子串拼接成一个新序列，要求新序列中每个数字最多出现一次。求满足条件的拼接序列的最大可能长度。

（注：若无法构造满足条件的非空序列，允许选择空串，此时长度为 $0$）

思路

典型的状态压缩dp。

我们发现 $a_i \le 18$ ，并且要求是要连续的 $a_i$ ，这就为我们的状压dp创造了时间条件。

于是我们可以定义状态 $f_i$ ，表示出现的集合状态为 $i$ 的情况下，所能得到的最大长度。

然后枚举每个位置，从当前位置开是枚举 $18$ 个，因为题目中要求是连续的 $a_i$ ，所以只需要枚举 $18$ 个即可。

接着我们可以通过 $f$ 求得 $dp_s$ ，表示出现状态为 $s$ 的子集时，所能得到的最大的长度。

从小到大枚举集合，然后再枚举 $s$ 的每一位进行 DP 转移即可。

答案就是 $\max { dp_s + dp_{m \oplus s}}$。
